1. Next.js 프레임워크
 - 기본구조가 정해져있음, 같이개발할 때 디렉터리구조, 파일구조 표준화되고
   사용하는 라이브러리들이 기본적으로 내장되어있음
 - 우리팀에서 서로 코드리뷰해줘야고 서로 통합할 때 같은 구조 개발할 수 있음
   Next.js 프레임워크

2. 페이지 파일 기반 라우팅 제공
 - Angular, React/Vue X
 - 페이지 폴더 및 파일구조만 보면 라우팅 구조를 파악할 수 있음
 - 새로운 라우팅 추가시에도 파일만 추가하면됨
   -> 공통적인 Route파일 수정 안해도 됨

3. SSR, CSR, 
 - SSR을 하더라도 전부 렌더링하지 않음, SEO필요하것만 렌더링함
 - 그 외의 부분들은 클라이언트에서 렌더링함
 - 대표적 Image -> alt속성과 1픽셀이미지를 렌더링하고, 
   이미지는 클라이언트에서 받아옴, 이미지크기 및 포맷최적화
   화면 영역 안쪽만 이미지를 로딩함, 스크롤되면 lazy 로딩함
 - 대표적 Link -> 웹페이지가 처음 로딩될 때는 SSR로딩(일단 컨텐츠가 먼저나옴)
             -> Link나, Router 이동할 때는 CSR로딩
                (페이지리프레시X, cSS/js리소스안받옴, 컴포넌트js만1회받아옴)

4. 풀스택 개발가능
 - API Routes(REST API) 작성 가능(기본)
 - 데이터베이스 연동가능(Prisma)
 - 클라이언트 및 서버 세션처리를 손쉽게 할 수 있음(NextAuth)
